## 1. エグゼクティブサマリー

「脳活ファミリー」は、3世代の家族が認知機能トレーニングを共に楽しむという独自のポジショニングを持つアプリである。本レビューでは、要件定義書、デザイン要件書、機能仕様書、UI設計方針書の4ドキュメントを横断的に分析した。

### 1.1 全体評価

| 観点 | 評価 | コメント |
|------|------|---------|
| 3世代ユーザビリティ | **A** | 3モード設計が秀逸。シニア向け配慮が特に手厚い |
| 感情設計 | **A+** | 赤色禁止、否定語排除、キャラクターの使い方が優秀 |
| オンボーディング | **B+** | 骨格は良いが、シニアの認知特性への更なる配慮が必要 |
| 認知負荷 | **B** | 機能の豊富さに対して情報設計の整理が不足 |
| エンゲージメント | **A-** | ゲーミフィケーションは充実。ただし一部にやりすぎリスク |
| アクセシビリティ | **A** | WCAG 2.1 AA/AAA対応が明確。細部に改善余地あり |

### 1.2 最も優れている点

1. **赤色完全排除の徹底**: エラー・警告をコーラル/オレンジで代替するという判断は、認知症予防アプリとして極めて正しい。シニアが「失敗した」と感じるトリガーを設計段階から排除している
2. **否定語排除と文言ガイドライン**: 「不正解」を「おしい!」に、「低スコア」を「伸びしろ」に言い換える文言設計は、認知症への不安を抱えるシニアの心理的安全性を高度に担保している
3. **3モード切替設計**: シニアモード/標準モード/キッズモードの設計が体系的で、フォントサイズ、タッチターゲット、アニメーション強度まで数値で定義されている点は実装品質を担保する

### 1.3 最も改善が必要な点

1. **オンボーディングのステップ数と認知負荷**: 5ステップは「数」としては少ないが、各ステップの認知的要求が高く、75歳以上のシニアには厳しい
2. **BottomNavの5タブ構成**: シニアにとって5つのタブは選択肢として多い。1画面1アクションの原則と矛盾する
3. **通知の洪水リスク**: 家族メンバー全員の活動通知が流れ込む設計は、親世代に「通知疲れ」を引き起こすリスクがある

## 2. 3世代ユーザビリティ

### 2.1 シニア世代 (60-90歳)

#### 評価: A-

**優れている点:**

| 項目 | 設計内容 | 評価 |
|------|---------|------|
| フォントサイズ | 最小20px、推奨22px | 適切。75歳以上の視力低下に対応 |
| タッチターゲット | 56px以上 | 優秀。指の震えにも対応可能 |
| コントラスト比 | 7:1以上 (AAA準拠) | 白内障にも配慮した高水準 |
| 音声ガイド | デフォルトON (75歳以上) | シニアプラスへの自動ON設定は良い判断 |
| ヒント機能 | 自動ヒント付きテスト | 「できない」体験を減らす設計 |
| 制限時間 | 1.3x-1.5xの余裕 | 焦りを軽減する配慮 |

**懸念事項:**

**(1) 操作ステップの3タップ以内到達について**

要件では「デイリーチャレンジ」がコア機能とされているが、現在の画面遷移を分析すると以下のステップが必要:

```
ホーム画面 → デイリーチャレンジ画面 → テストプレイ画面
```

これは3タップだが、テストプレイ画面に到達するまでに「チャレンジ開始ボタン」のタップが必要であり、実質4タップとなる。シニアにとって、ホーム画面から直接テストに入れる「ワンタップ開始」ボタンが望ましい。

**(2) BottomNavの認知負荷**

5タブ構成（ホーム / ファミリー / キャラ / レポート / 設定）は、シニアにとって選択肢が多すぎる。

```
現在: ホーム | ファミリー | キャラ | レポート | 設定 (5タブ)
提案: ホーム | かぞく | きろく (3タブ) + 設定はハンバーガーメニューへ
```

シニアモードではBottomNavを3タブに減らし、「キャラクター」と「レポート」はホーム画面内のカードからアクセスさせるべきである。

**(3) QRコードスキャンの難易度**

ファミリーグループ参加のQRコードスキャンは、75歳以上のシニアには技術的ハードルが高い。「息子に見せてもらったQRコードを自分のスマホで読む」という行為は、2台のデバイスの同時操作を要求する。招待コード（6桁英数字）方式がシニアにとっての主導線であるべきで、QRは補助手段に位置づけるべきである。

**(4) パスワード入力の困難さ**

シニア向けに「ログイン状態を長期間維持」は良い判断だが、初回のパスワード入力自体が大きなハードルとなる。パスワードレス認証（マジックリンク方式やSMSワンタイムコード）を主要な認証手段として検討すべきである。

#### シニア向け改善提案サマリー

| ID | 提案 | 優先度 |
|----|------|--------|
| S-01 | ホーム画面に「今日のトレーニング開始」ワンタップボタンを配置 | P0 |
| S-02 | シニアモードではBottomNavを3タブに削減 | P1 |
| S-03 | パスワードレス認証（マジックリンク/SMSコード）の追加 | P1 |
| S-04 | QRコードより招待コード入力を主導線に変更（シニア向けUI） | P2 |
| S-05 | 緊急連絡先への自動通知機能（長期未アクセス時） | P2 |

### 2.2 親世代 (30-60歳)

#### 評価: A-

**優れている点:**

| 項目 | 設計内容 | 評価 |
|------|---------|------|
| ファミリーダッシュボード | カード形式での一覧表示 | 一目で家族の状況を把握できる |
| 「今日の調子」アイコン | 太陽/晴れ時々曇り/曇り | 直感的で赤信号を避けた設計 |
| トレンドアラート | やさしい文面での通知 | 過度に不安を煽らない配慮 |
| 月次レポートPDF | 医師への共有が可能 | 実用的な機能 |

**懸念事項:**

**(1) 見守り機能のアクセス導線**

親世代の最大の利用動機は「離れた親の健康状態の確認」だが、現在の設計ではこの機能はBottomNavの「ファミリー」タブ配下にある。ホーム画面のファーストビューに「家族の今日の状況」サマリーカードを配置し、タップでダッシュボードに遷移させるべきである。

**(2) 通知の粒度設定が不十分**

通知設定は「ON/OFF」のみの記載が多く、粒度の調整が不足している。親世代は「シニアの未実施通知は受けたいが、子供の通常テスト完了通知は不要」といった細かい制御を求める。

```
提案: 通知設定を以下の粒度で提供
├── 見守り対象別ON/OFF（人ごと）
├── 通知種別別ON/OFF（テスト完了/未実施/アラート/メッセージ）
├── 時間帯制限（22:00-7:00は通知しない等）
└── 要約モード（個別通知ではなく1日1回のダイジェスト）
```

**(3) 医師共有フローの改善**

月次レポートのPDFダウンロードは良い機能だが、「かかりつけ医と共有する」ユースケースを考えると、共有リンク生成（閲覧権限付き、期限付きURL）の方が実用的である。PDFはメールやメッセンジャーで送る手間がある。

#### 親世代向け改善提案サマリー

| ID | 提案 | 優先度 |
|----|------|--------|
| P-01 | ホーム画面に「家族の今日の状況」サマリーカードを配置 | P0 |
| P-02 | 通知設定を人別/種別/時間帯で粒度調整可能に | P1 |
| P-03 | 月次レポートの共有リンク生成機能を追加 | P2 |
| P-04 | ダイジェスト通知モード（1日1回まとめ通知）を追加 | P2 |

### 2.3 子供・孫世代 (5-30歳)

#### 評価: B+

**優れている点:**

| 項目 | 設計内容 | 評価 |
|------|---------|------|
| アニメーション強度 | 150%で派手な演出 | 子供のエンゲージメントに効果的 |
| ふりがな対応 | 小学生以下に漢字のルビ付き | 学年に応じた適切な対応 |
| のうかつくん | キャップ、リュック、元気なポーズ | 子供に親しみやすいキャラクター |
| 世代対抗戦 | おばあちゃんチーム vs キッズチーム | 世代間交流の良い仕掛け |

**懸念事項:**

**(1) 5-7歳と8-12歳の体験差の不足**

現在「キッズモード」は5-12歳をひとまとめにしているが、5歳児と12歳児の認知能力・操作能力の差は極めて大きい。5-7歳は文字の読み書きが不完全であり、ひらがなの読解すら難しい場合がある。

```
提案: キッズモードを2段階に分割
├── プレキッズ (5-7歳): 音声ガイド必須、文字最小化、イラスト中心UI
└── キッズ (8-12歳): 現在のキッズモード設計
```

**(2) ゲーム要素のバリエーション不足**

テスト種別は13種類あるが、すべて「問題を解く」形式である。子供世代には「育成要素」「コレクション要素」「対戦要素」が重要なエンゲージメント要因となる。キャラクター育成（のうたんの成長段階）は良い設計だが、着せ替え以外のカスタマイズ要素が少ない。

**(3) 年齢制限の設計（COPPA準拠）**

13歳未満のアカウント作成に保護者の同意フローが記載されているが、同意フローの具体的なUI設計が不足している。保護者のデバイスで同意してから子供のデバイスに切り替える、というフローは複雑になりがちで、設計の詳細化が必要。

**(4) 10代後半-20代のモチベーション設計**

「ジュニア (13-19歳)」と「スタンダード (20-59歳)」の区分において、13-25歳前後のユーザーが「おばあちゃんのためのアプリ」と感じてしまうリスクがある。この世代には「自分の脳力を競う」という自己成長モチベーションを前面に出す設計が必要。

#### 子供・孫世代向け改善提案サマリー

| ID | 提案 | 優先度 |
|----|------|--------|
| C-01 | プレキッズモード (5-7歳) の新設 | P1 |
| C-02 | 10代後半-20代向けの「脳力ランキング」要素の強化 | P2 |
| C-03 | COPPA準拠の保護者同意フローの詳細UI設計 | P1 |
| C-04 | ミニゲーム的な追加コンテンツ（パズル、クイズバトル等） | P3 |

## 3. 感情設計

### 3.1 全体評価: A+

感情設計はこのアプリの最大の強みであり、設計者の深い洞察が反映されている。

### 3.2 スコア低下時の表現

#### 評価: A+

**現在の設計:**

| 場面 | 表現 | 評価 |
|------|------|------|
| 低スコア | 「ここが伸びしろだよ」 | 優秀。否定を完全に排除 |
| スコア下降トレンド | オレンジ矢印（赤は不使用） | 適切。視覚的な不安を軽減 |
| テスト不正解 | 「おしい! もう一度やってみよう」 | 優秀。挑戦意欲を維持 |
| ストリーク途切れ | 「今日からまた始めよう!」 | 優秀。罪悪感を与えない |
| 未達成 | 「あと少しで目標達成!」 | 優秀。進捗にフォーカス |

**追加検討が必要な場面:**

しかし、以下の場面についてはまだ設計が不足している:

| 場面 | 現状 | リスク | 提案 |
|------|------|--------|------|
| 月次レポートで3ヶ月連続低下 | 設計なし | 深刻な不安を引き起こす可能性 | 「体調や睡眠の影響もあります。ご自身のペースで大丈夫ですよ」と補足メッセージ |
| 家族ランキングで常に最下位 | 順位表示 | 劣等感、参加意欲の低下 | 「個人ベスト更新」にフォーカスしたビュー切替を優先表示 |
| 年齢別平均比較で下位 | 偏差値的表示 | 「自分は衰えている」との認識 | 平均以下の場合は比較自体を非表示にし、「自分の成長推移」のみ表示 |
| 難易度が自動降格 | レベルDOWN通知 | 能力低下の実感 | 降格を通知しない。自然に問題が簡単になるだけにする |

### 3.3 赤色禁止ルールの徹底度

#### 評価: A

赤色禁止は設計書全体で一貫して徹底されている。特に以下の点が優れている:

- エラー状態を `#E8845C`（コーラル）で代替
- 下降トレンドをオレンジ矢印で表示（赤矢印ではなく）
- 「危険」セマンティックカラーもコーラル系

**唯一の懸念:**

仕様書の早期警告システム（F-007-02）の警告レベルに「要確認（目立たない赤）」という記述がある。これはデザイン要件の赤色禁止ルールと矛盾している。この「目立たない赤」をオレンジまたは濃いコーラルに統一すべきである。

### 3.4 キャラクターの励ましタイミング

#### 評価: A-

キャラクター（のうかつ博士/先生/くん）の3バリアント設計は秀逸。世代別の口調差も自然で違和感がない。

**優れている点:**
- テスト前、正解時、不正解時、ストリーク達成時、空状態、エラー画面のすべてに台詞パターンが定義されている
- 不正解時に「考え中の表情（優しいまま）」を維持する設計は、心理的安全性の確保に直結する
- 台詞のランダム表示による新鮮さの維持

**改善が必要な点:**

| 場面 | 懸念 | 提案 |
|------|------|------|
| テスト中のアイドル状態 | 長時間考え込むシニアに対する声かけがない | 20秒以上回答がない場合「ゆっくり考えて大丈夫ですよ」の表示 |
| アプリ復帰時 | 3日以上未起動後の復帰時の声かけが不明確 | 「おかえりなさい! 待っていたよ」など、責めずに歓迎する台詞 |
| 難易度降格後 | 降格に対するキャラクターの反応が未定義 | 問題が簡単になったことを「今日はリラックスモードだよ」と自然にカバー |

### 3.5 否定語排除の徹底度

#### 評価: A+

UI設計方針書の7.3節「否定形を使わないルール」は極めて充実しており、7.6節の「医療用語の言い換え表」も実用的である。

特に以下の言い換えが秀逸:
- 「認知機能低下」 → 使わない（代わりに「伸びしろ」）
- 「検査」 → 「トレーニング」「チャレンジ」
- 「予防」 → 「元気を保つ」「脳を鍛える」

**追加すべき言い換え:**

| 使ってはいけない表現 | 言い換え |
|-------------------|---------|
| タイムアウト | 「時間です! お疲れさまでした」 |
| ゲームオーバー | 使わない |
| 残りライフ | 使わない |
| ペナルティ | 使わない |
| 再試行 | 「もう一度チャレンジ」 |
| 制限 | 「今日の分」 |
| 警告 | 「お知らせ」 |

## 4. オンボーディング

### 4.1 全体評価: B+

### 4.2 初回起動から最初のテスト完了まで

**現在の設計 (F-001-04):**

```
Step 1: 「こんにちは!」アプリの目的説明（30秒）
Step 2: お名前入力（キーボード or 音声入力）
Step 3: 生年月日選択（大きなドラムロール式ピッカー）
Step 4: ファミリーグループ参加（QRスキャン or コード入力）
Step 5: 初回テスト体験（お試し3問）
```

**時間見積もり:**

| ステップ | 想定所要時間 (シニア) | 想定所要時間 (標準) |
|---------|---------------------|-------------------|
| Step 1 | 30-60秒 (音声ガイド聴取) | 15-30秒 |
| Step 2 | 30-90秒 (キーボード操作) | 10-20秒 |
| Step 3 | 30-60秒 (ドラムロール操作) | 10-20秒 |
| Step 4 | 60-180秒 (QR/コード) | 30-60秒 |
| Step 5 | 120-180秒 (テスト体験) | 60-120秒 |
| **合計** | **4.5-9.5分** | **2-4分** |

シニアの場合、5分以内の完了は厳しい。特にStep 4のファミリーグループ参加は、QRスキャンの技術的ハードルとコード入力の操作的ハードルが高く、この段階で離脱する可能性が極めて高い。

### 4.3 シニアオンボーディングの改善提案

**提案: 「まず体験、後から設定」モデル**

```
改善案:

Step 1: 「こんにちは!」+ お名前入力（音声入力推奨）
         ↓
Step 2: 生年月日選択（年代を大きなボタンで選ぶ形式）
         ↓
Step 3: お試しテスト1問（最もシンプルな絵合わせ）
         ↓
        「やったね! もっとやってみたい?」
         ↓
Step 4: ホーム画面到達（ファミリー参加は後から促す）

※ ファミリーグループ参加はホーム画面の目立つカードから
  「家族を招待してみよう!」として段階的に誘導
```

**改善のポイント:**

1. **生年月日のドラムロール → 年代ボタン**: 75歳のシニアに西暦のドラムロールを操作させるのは困難。「70代」「80代」などの大きなボタンで年代を選択させ、その後に生まれ年を選ぶ2段階にする
2. **ファミリー参加のスキップ**: オンボーディングの最大のボトルネック。Step 4をオンボーディングから外し、ホーム画面到達後に促す
3. **テスト体験の前倒し**: 「このアプリは楽しい」という体験を最速で提供する
4. **3ステップ完了**: シニアの認知能力で安全に完了できる最小ステップ数

### 4.4 ファミリー招待フローの改善

現在の3方式（QRコード/招待リンク/招待コード）は網羅的だが、「誰が誰を招待するのか」のユースケース整理が不足している。

**最も多いユースケース:**

```
息子(太郎) が 母(花子) をアプリに招待する
    │
    ├── 同席時: 太郎のスマホでQRを表示 → 花子がスキャン
    │   → 問題: 花子がQRスキャンできない可能性
    │   → 改善: 太郎が花子のスマホを操作して参加させる「代理登録」機能
    │
    └── 遠隔時: 太郎がLINEで招待リンクを送る → 花子がリンクをタップ

        → 改善: 電話で「○○○○○○（6桁）を入力してね」と伝えるフローを推奨
```

**提案: 「代理登録」機能**

家族（主に息子/娘）がシニアのスマホを操作して、代理でアカウント作成とファミリーグループ参加を完了できる機能。U.S.-007のユーザーストーリーでも「息子の太郎さんにアプリをインストールしてもらった」と記載されており、この現実に即した機能である。

### 6.3 家族間インタラクションの自然さ

#### 評価: B+

**優れている点:**

- スタンプによるワンタップコミュニケーション: テスト結果画面から直接送信できる設計は、コミュニケーションの摩擦を最小化している
- ボイスメッセージ: シニアがキーボード入力せずに気持ちを伝えられる
- 自動通知: テスト完了やバッジ獲得の自動共有で、自然な会話のきっかけを創出

**懸念事項:**

**(1) インタラクションの一方向性**

現在の設計では、インタラクションの多くが「通知/共有 → リアクション」の一方向。家族間の双方向的なインタラクションが不足している。

```
提案: 双方向インタラクション
├── 「一緒にテスト」機能: 同時刻に同じ問題をリアルタイムで解く
├── 「お題出し」機能: 家族が互いにテスト問題を選んで出し合う
└── 「応援リクエスト」: テスト開始時に家族に「応援してね」と通知
```

**(2) メッセージの時間軸管理**

メッセージの保存期間がスタンプは無制限、ボイスは30日と設定されているが、メッセージ画面のUI上で「古いメッセージ」と「新しいメッセージ」の区別がどう表現されるかの設計が不足。時間軸でグルーピングし、「今日」「昨日」「先週」などの区切りを入れるべき。

### 6.4 ゲーミフィケーションのバランス

#### 評価: A-

**バランスの良い点:**

- バッジが5カテゴリ（継続系/スコア系/ファミリー系/季節系/特別系）に分かれており、多様な行動を促進する
- コインの用途がキャラクター着せ替えに限定されており、課金圧や射幸心を避けている
- 連続記録が途切れても過去最高記録は保持される設計は、シニアの心理的安全性に配慮

**やりすぎリスク:**

| リスク | 内容 | 対策提案 |
|--------|------|---------|
| 通知の洪水 | 家族5人がそれぞれバッジ獲得・テスト完了すると大量通知 | 1日5通制限は良いが、「まとめ通知」オプションも追加 |
| ストリーク強迫 | 連続記録を維持するためだけにアプリを開く行動 | 「お休みチケット」(月2回、連続記録を途切れさせずに休める) を導入 |
| 競争ストレス | ファミリーランキングが一部ユーザーにプレッシャー | ランキングのON/OFF設定。デフォルトOFFでオプトイン |
| コイン獲得の焦り | 限定アイテムの期間終了前にコインが足りない | 期間終了後も割増価格で購入可能にする |

**「お休みチケット」の詳細設計案:**

```
お休みチケット:
├── 毎月2枚自動付与
├── 使用: テスト未実施日の翌日に自動消費（遡及）
├── 表示: カレンダーに「お休みマーク」(お布団アイコン)
├── のうたん台詞: 「ゆっくり休むのも大切だよ!」
└── 効果: 連続記録が途切れない
```

### 7.3 色覚多様性対応

#### 評価: B+

**現在の対応:**

- 色だけに依存しない情報伝達（パターン・形状の併用）が原則として記載
- グラフの凡例にパターン（破線、点線、実線）を併用

**改善が必要な点:**

**(1) 世代別アクセントカラーの色覚シミュレーション**

3つの世代カラー（ティール/ブルー/コーラルピンク）は、P型色覚（赤色覚異常）のユーザーにとってティールとブルーの区別が困難になる可能性がある。

```
対策提案:
├── 色に加えて、アイコンの形状で世代を識別（丸/四角/星）
├── アバター枠の線種を変える（実線/破線/二重線）
└── 色覚シミュレーションツールでの全パターン検証を実装チェックに追加
```

**(2) ストループテストの配慮**

ストループテストは「色」を判定する課題であり、色覚多様性のユーザーには本質的に不公平。代替テスト（形状版ストループ等）を用意するか、色覚設定に基づいて使用色を調整する機能が必要。

### 7.4 音声読み上げ対応

#### 評価: A-

**現在の対応:**

- VoiceOver (iOS) / TalkBack (Android) の完全対応を目標
- `aria-live` によるスコア更新・タイマーの読み上げ
- テスト問題の読み上げ対応

**改善が必要な点:**

**(1) テストゲームのスクリーンリーダー対応**

以下のテストはビジュアル中心であり、スクリーンリーダーでの操作が本質的に困難:

| テスト | 課題 | 代替案 |
|--------|------|--------|
| 絵合わせ（神経衰弱） | カードの位置を音声で伝えるのが困難 | 音声で「左上」「右下」と位置を読み上げ + 図柄の音声説明 |
| 間違い探し | ビジュアル比較が前提 | 代替テスト（音の間違い探し等）を提供 |
| ジグソーパズル | ドラッグ&ドロップが前提 | 代替テスト（順序並べ替え等）を提供 |
| 図形回転 | 視覚的な空間把握が前提 | 代替テスト（言語的な空間記述問題）を提供 |

**(2) 音声ガイドの設計詳細**

「音声ガイド付き」と記載されているが、具体的に何を読み上げるのか、読み上げのタイミング、速度調整、一時停止/再生の操作方法が不足している。

```
提案: 音声ガイドの詳細仕様
├── 読み上げ速度: 通常 / ゆっくり / とてもゆっくり の3段階
├── 一時停止: 画面タップで一時停止/再開
├── 繰り返し: 「もう一度聞く」ボタン
├── 自動/手動: 自動読み上げ or 「読み上げ」ボタンタップ
└── 対象: テスト問題文、選択肢、結果、ナビゲーション
```

### 7.5 運動機能への配慮

#### 評価: B+

**対応済み:**

- タッチターゲット最小44px (シニア56px)
- ボタン間隔最小8px (シニア12px)

## 9. まとめ

### 9.1 総括

「脳活ファミリー」の設計書群は、認知症予防アプリとして極めて高い品質を持つ。特に感情設計（赤色禁止、否定語排除、ポジティブ表現の徹底）は業界水準を大きく超えており、ユーザーの心理的安全性への配慮が設計の隅々まで行き届いている。

3モード切替設計（シニア/標準/キッズ）は、3世代が同じアプリを使うというユニークな要件に対する構造的に正しい回答であり、各モードのパラメータが数値で定義されている点は実装品質の担保につながる。

### 9.2 最優先で取り組むべきこと

1. **シニアのオンボーディング簡素化**: ファミリー参加をオンボーディングから分離し、3ステップでホーム画面に到達させる
2. **シニアモードのホーム画面簡素化**: 「今日のトレーニング開始」ボタンを画面の主役にする
3. **難易度降格の非通知化**: 能力低下の実感を与えない設計に変更
4. **年齢別平均比較の条件付き非表示**: 平均以下のユーザーには個人の成長推移のみ表示

### 9.3 設計書間の整合性に関する注記

以下の設計書間の不整合を発見したため、Phase 4（実装）前に解消すべきである:

| 箇所 | 不整合内容 | 解消方針 |
|------|----------|---------|
| requirements.md (F-006-01) vs design-requirements.md (4.3) | キャラクター名が「のうたん」と「のうかつ博士」で揺れている | デザイン要件書の「のうかつ博士」（3バリアント）に統一。「のうたん」は愛称として使用可能 |
| requirements.md (F-007-02) | 早期警告の「要確認（目立たない赤）」がデザイン要件の赤色禁止と矛盾 | オレンジまたは濃いコーラルに変更 |
| requirements.md (F-008-01) | 文字サイズ設定の「小: 14px」がデザイン要件の最小16px (標準モード) と矛盾 | 「小」を16pxに引き上げ |
| spec.md (3.1.3) | シニアプラスのタッチターゲット64pxがデザイン要件の56pxと異なる | 64pxをシニアプラスの推奨値として採用（より安全） |
