## 1. 基本方針

### モバイルファースト

**スマートフォンが主要ターゲットデバイス**。シニア世代のメインデバイスがスマートフォンであるため、モバイルでの体験を最優先に設計する。タブレット・デスクトップはプログレッシブエンハンスメントで対応する。

### 設計原則

| 原則 | 説明 |
|------|------|
| コンテンツファースト | 画面サイズに関わらず情報の優先度は同じ |
| フレキシブル | 固定幅レイアウトは禁止。相対単位（%, rem, vw）を使用 |
| タッチフレンドリー | すべてのインタラクティブ要素はタッチ操作に最適化 |
| 段階的開示 | 小画面では情報を折りたたみ、必要に応じて展開 |
| モード統合 | 画面サイズ x UIモード（シニア/標準/キッズ）の組み合わせを考慮 |

### ビューポート設定

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=5.0" />
```

- `maximum-scale=5.0`: シニアユーザーのピンチズームを許可（`user-scalable=no` は禁止）
- `initial-scale=1.0`: 初期表示を最適サイズに

### ビューポート高さ

```css
/* h-dvh を使用。h-screen は禁止 */
.full-height {
  height: 100dvh; /* Dynamic Viewport Height */
}
```

- `h-screen` は使用禁止（iOS Safari のアドレスバーでコンテンツが隠れるため）
- `100dvh` を使用してアドレスバーの表示/非表示に追従

## 2. ブレークポイント定義

### Tailwind CSS ブレークポイント

| トークン | 幅 | 対象デバイス | 用途 |
|---------|-----|------------|------|
| (default) | 0-639px | スマートフォン | ベースデザイン |
| `sm` | 640px~ | 大型スマートフォン | 微調整 |
| `md` | 768px~ | タブレット（縦） | 2カラムレイアウト開始 |
| `lg` | 1024px~ | タブレット（横）/ 小型PC | フルレイアウト |
| `xl` | 1280px~ | デスクトップ | ワイドレイアウト |
| `2xl` | 1536px~ | 大型デスクトップ | 最大幅制限 |

### コンテナ幅

```css
.container {
  width: 100%;
  max-width: 1280px; /* xl */
  margin: 0 auto;
  padding: 0 16px;
}

@media (min-width: 640px) {
  .container { padding: 0 24px; }
}

@media (min-width: 1024px) {
  .container { padding: 0 32px; }
}
```

### コンテンツ最大幅

| コンテンツタイプ | 最大幅 | 理由 |
|---------------|--------|------|
| テキストコンテンツ | 720px | 可読性（1行60-80文字） |
| カードグリッド | 1200px | 3カラム表示に十分 |
| フルワイド | 1536px | 大画面対応 |
| テスト画面 | 640px | 集中しやすい狭い幅 |

## 3. グリッドシステム

### 基本グリッド

8px グリッドシステムをベースとする。

```css
/* スペーシングは8の倍数 */
:root {
  --grid-unit: 8px;
}
```

### カラムグリッド

| 画面サイズ | カラム数 | ガター | マージン |
|----------|---------|--------|--------|
| モバイル (~639px) | 4 | 16px | 16px |
| タブレット (640-1023px) | 8 | 24px | 24px |
| デスクトップ (1024px~) | 12 | 24px | 32px |

### Tailwind グリッド使用例

```tsx
// カードグリッド: モバイル1列 → タブレット2列 → デスクトップ3列
<div className="grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3">
  <GameCard />
  <GameCard />
  <GameCard />
</div>

// テスト結果: モバイル縦積み → タブレット横並び
<div className="flex flex-col gap-6 md:flex-row">
  <ScoreCard className="md:flex-1" />
  <RadarChart className="md:flex-1" />
</div>
```

## 4. タッチターゲット

### モード別タッチターゲットサイズ

| モード | 最小サイズ | 推奨サイズ | タップ間隔 |
|--------|----------|----------|----------|
| 標準モード | 44px x 44px | 48px x 48px | 8px以上 |
| シニアモード | 48px x 48px | 56px x 56px | 12px以上 |
| キッズモード | 48px x 48px | 52px x 52px | 8px以上 |

### 実装ルール

```css
/* すべてのインタラクティブ要素に適用 */
.interactive {
  min-width: 44px;
  min-height: 44px;
  /* ヒットエリア拡張（視覚サイズを超えてタップ可能に） */
  position: relative;
}
.interactive::after {
  content: "";
  position: absolute;
  inset: -4px; /* 周囲にヒットエリア追加 */
}

/* シニアモード */
[data-mode="senior"] .interactive {
  min-width: 56px;
  min-height: 56px;
}

/* キッズモード */
[data-mode="kids"] .interactive {
  min-width: 52px;
  min-height: 52px;
}
```

### アイコンのみボタンのタッチエリア

アイコンボタン（閉じるボタン等）は視覚的に小さくても、タップエリアを広く確保する。

```tsx
// 24pxアイコンでも44px以上のタップエリアを確保
<button className="relative flex items-center justify-center w-10 h-10 senior:w-14 senior:h-14">
  <X className="h-6 w-6 senior:h-8 senior:w-8" />
</button>
```

## 8. シニアモードのレスポンシブ特別対応

### 原則: 1画面1アクション

シニアモードでは画面サイズに関わらず、情報密度を低く保ち、常に「今何をすべきか」が明確であることを保証する。

### モバイル x シニアモード

| 項目 | 標準モードとの違い |
|------|--------------|
| コンテンツ幅 | 左右マージンを 24px に拡大（標準: 16px） |
| カード間隔 | 24px（標準: 16px） |
| ボタン高さ | 56px以上（標準: 44px） |
| フォントサイズ | 最小 20px（標準: 16px） |
| 行間 | 1.8（標準: 1.6） |
| 1画面情報量 | 最大3アクション |
| スクロール量 | 最小限。重要情報はファーストビューに |
| ボトムナビ高さ | 80px（標準: 64px） |
| ナビラベル | 常に表示、16px |

### タブレット x シニアモード

| 項目 | 仕様 |
|------|------|
| カラム数 | 1-2カラム（標準は2-3） |
| カード | 大きめ表示、情報量を減らす |
| グラフ | データポイント最大7、ラベル大きめ |
| タブ | 大きいタブ (56px高)、アイコン+テキスト常時 |

### デスクトップ x シニアモード

| 項目 | 仕様 |
|------|------|
| サイドナビ幅 | 280px（標準: 240px） |
| ナビアイコン | 32px（標準: 24px） |
| ナビテキスト | 20px（標準: 16px） |
| メインコンテンツ | 最大幅 1000px（標準: 1200px） |
| カード | 2カラムまで（標準: 3カラム） |

### シニアモード固有UI要素

```
┌────────────────────────────┐
│  🔙 もどる       ホーム画面   │  ← 常に表示される「戻る」ボタン (大きめ)
├────────────────────────────┤
│                            │
│   ┌──────────────────┐    │
│   │                  │    │
│   │   今日のテスト      │    │  ← 1画面1アクション
│   │                  │    │
│   │   [テストをはじめる]  │    │  ← 大きなボタン (56px)
│   │                  │    │
│   └──────────────────┘    │
│                            │
│   のうかつ博士:              │  ← キャラクターメッセージ
│   「今日もがんばりましょう!」   │     (20px、丁寧語)
│                            │
├────────────────────────────┤
│ 🏠 ホーム  🧠 テスト  👥 家族 │  ← 大きなナビ (80px)
└────────────────────────────┘
```

### シニアモード: 操作確認の強化

```
通常モード:
  ボタンタップ → 即実行

シニアモード:
  ボタンタップ → 確認ダイアログ → 実行
  （データ削除、設定変更、送信など重要操作）
```

### シニアモード: テキスト入力の補助

| 補助機能 | 仕様 |
|---------|------|
| 自動フォーカス | フォーム表示時に最初の入力欄に自動フォーカス |
| 入力候補 | よく使う値を候補として表示 |
| 音声入力 | 音声入力ボタンを目立つ位置に配置 |
| エラー表示 | 入力欄のすぐ下に、大きな文字でエラー内容を表示 |
| クリアボタン | 入力欄内に大きなクリアボタン |

| 用途 | フォーマット | フォールバック |

| アイコン | Lucide React (SVGインライン) | - |

| 画面サイズ | キャラクターサイズ | 用途 |

## 12. タイポグラフィのレスポンシブ

### フォントサイズの流動スケーリング

基本的にモード別の固定サイズを使用するが、見出し (Display, H1) のみ `clamp()` で流動的にスケーリングする。

```css
/* Display: モバイル 32px → デスクトップ 40px */
.display {
  font-size: clamp(2rem, 1.5rem + 2vw, 2.5rem);
}

  font-size: clamp(2.5rem, 2rem + 2vw, 3rem);
}
```

## 14. パフォーマンス考慮

### デバイス別パフォーマンス戦略

| デバイスカテゴリ | 戦略 |
|--------------|------|
| ローエンドスマホ（シニア多い） | アニメーション最小化、画像軽量化、JS最小限 |
| ミドルレンジスマホ | バランスのとれた体験 |
| タブレット / デスクトップ | フル体験 |

### シニアデバイス特別考慮

シニアユーザーは古いデバイスを使用していることが多いため、以下を考慮する。

| 項目 | 対策 |
|------|------|
| `backdrop-filter` | 使用最小限。フォールバック背景色を用意 |
| `blur()` | 10px以下に制限 |
| `will-change` | アニメーション直前にのみ付与 |
| アニメーション | シニアモードでは30%に減量。`transform`/`opacity` のみ |
| 画像 | 適切な `srcset` で小さいサイズを配信 |
| フォント | サブセット化でファイルサイズ削減 |
| JS bundle | コード分割で初期ロードを軽量化 |

### Core Web Vitals 目標

| 指標 | 目標値 | 対策 |
|------|--------|------|
| FCP | < 1.8s | フォント `font-display: swap`、キャラクター画像プリロード |
| LCP | < 2.5s | 重要画像のプリロード、SSR/SSG |
| CLS | < 0.1 | フォント読み込みのCLS防止、画像サイズ指定 |
| INP | < 200ms | タップの即時応答、重い処理のWeb Worker委譲 |

### CLS 防止策

```tsx
// 画像の CLS 防止: 必ず width/height を指定
<img src="..." width={300} height={200} alt="..." />

// フォントの CLS 防止: font-display: swap + フォールバックサイズ調整
@font-face {
  font-family: "Zen Maru Gothic";
  font-display: swap;
  size-adjust: 100%;
  ascent-override: 90%;
}
```

## 付録

### A. レスポンシブ設計のTailwind CSS早見表

```tsx
// 基本パターン: モバイル → sm → md → lg → xl
className="
  p-4            // モバイル: padding 16px
  sm:p-5         // 640px+: padding 20px
  md:p-6         // 768px+: padding 24px
  lg:p-8         // 1024px+: padding 32px

  grid-cols-1    // モバイル: 1列
  sm:grid-cols-2 // 640px+: 2列
  lg:grid-cols-3 // 1024px+: 3列

  text-base      // モバイル: 16px
  md:text-lg     // 768px+: 18px
  lg:text-xl     // 1024px+: 20px
"
```

### B. モード x レスポンシブの複合クラス

```tsx
// シニアモード x モバイル → シニアモード x デスクトップ
className="
  // 標準モード ベース
  min-h-[44px] text-base p-4

  // レスポンシブ
  md:p-6 lg:p-8

  // シニアモード
  senior:min-h-[56px] senior:text-[20px] senior:p-6
  senior:md:p-8 senior:lg:p-10

  // キッズモード
  kids:min-h-[48px] kids:text-[18px]
"
```

### C. Safe Area 対応

```css
/* iOS のノッチ・ホームバー対応 */
.bottom-nav {
  padding-bottom: env(safe-area-inset-bottom);
}

.main-content {
  padding-left: env(safe-area-inset-left);
  padding-right: env(safe-area-inset-right);
}

/* ヘッダー (Dynamic Island 対応) */
.header {
  padding-top: env(safe-area-inset-top);
}
```

### D. 禁止レイアウトパターン

| 禁止パターン | 理由 | 代替 |
|------------|------|------|
| `h-screen` | iOS Safari アドレスバー問題 | `h-dvh` |
| `user-scalable=no` | シニアのズームを妨げる | 削除 |
| `position: fixed` のみの全画面 | スクロール不能 | `h-dvh` + `overflow-auto` |
| 横スクロール必須のテーブル | シニアが操作困難 | カード形式に分解 |
| 無限スクロール | 現在位置を見失う | ページネーション |
| 自動カルーセル | 操作が追いつかない | 手動スワイプのみ |
| パララックススクロール | シニアに酔いを与える | 静的背景 |
